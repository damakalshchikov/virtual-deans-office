# Виртуальный деканат - Управление практикой студентов

## Описание

Веб-приложение «Виртуальный деканат» автоматизирует полный цикл управления
производственной практикой студентов: от планирования и распределения по организациям
до аттестации и формирования итоговых документов.

## Архитектура (6 модулей)

| Модуль | Описание |
|--------|----------|
| **Планирование** | Создание практик, этапов, назначение руководителей |
| **Распределение** | Вакансии, заявки, matching по компетенциям, распределение |
| **Выполнение** | Задания, сдача результатов, проверка куратором/преподавателем |
| **Аттестация** | Отчёты, дневники, характеристики (с просмотром текста и вложениями), аттестационные листы, комиссии |
| **Администрирование** | Пользователи, роли, права, справочники (кафедры, организации, компетенции) |
| **Сервисный** | Профиль, компетенции студента, система уведомлений |

## Стек технологий

- **Backend:** Python 3, Flask 3.1, SQLAlchemy, Flask-Login
- **Frontend:** Bootstrap 5.3, Font Awesome 6.5, Jinja2
- **СУБД:** PostgreSQL

## Установка и запуск

```bash
# 1. Создать и активировать виртуальное окружение
python -m venv .venv
source .venv/bin/activate        # Linux / macOS
# .venv\Scripts\activate         # Windows

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Применить схему БД (один раз, требует PostgreSQL)
psql -U postgres virtual_dekanat < schema.sql

# 4. Загрузить тестовые данные (пересоздаёт все таблицы)
python seed.py

# 5. Запустить сервер
flask --app app:create_app run --debug
```

Приложение будет доступно по адресу: http://localhost:5000

Для переопределения строки подключения к БД:

```bash
DATABASE_URL="postgresql://user:pass@host/dbname" flask --app app:create_app run --debug
```

## Ролевая модель доступа (RBAC)

- **Администратор:** полный доступ ко всем модулям
- **Преподаватель:** планирование практик, создание заданий, аттестация, распределение
- **Куратор:** управление вакансиями, проверка заданий, написание характеристик
- **Студент:** заявки на вакансии, выполнение заданий, подача отчётов и дневников

## Загрузка файлов

В модуле **Выполнение** можно прикреплять файлы к комментариям задания.
В модуле **Аттестация** можно прикреплять файлы к отчётам, дневникам и характеристикам.

Допустимые форматы: PDF, DOC, DOCX, TXT, PNG, JPG, ZIP, XLSX, XLS. Максимальный размер — 16 МБ.

## Алгоритм matching компетенций

При подаче заявки на вакансию система автоматически рассчитывает процент
совпадения компетенций студента с требованиями вакансии:

```
match% = (число компетенций, где уровень студента >= минимального) / (число требований) * 100
```

Если у вакансии нет требований, возвращается 100%.
