## 2. Роли пользователей системы

Роли определены на основе диаграмм IDEF0 и DFD всей команды. Каждому пользователю назначается **одна роль** (статическое разделение обязанностей).

| Роль | Субъект системы | Область ответственности в ЛП3 |
|------|----------------|-------------------------------|
| `student` | Студент | Просматривает свои задания и статус их выполнения |
| `curator` | Руководитель от организации | Видит закреплённых студентов и их прогресс |
| `teacher` | Руководитель от кафедры | Наблюдает за ходом практики студентов |
| `admin` | Деканат / администратор | Управляет пользователями и их ролями |

## 3. Что необходимо реализовать

### Часть 1 - Идентификация и аутентификация

**Выбранный метод:** постоянные многократно используемые пароли.

**Обоснование выбора:** система «Виртуальный деканат» - внутренняя учебная ИС без повышенных требований к безопасности. Постоянные пароли с надёжным хешированием (bcrypt) оптимальны по соотношению сложность/практичность.

**Что реализуется:**

1. **Страница входа `/login`**
   - Форма: поле «Логин» + поле «Пароль»
   - При отправке: поиск пользователя в БД по логину → проверка пароля через `check_password_hash`
   - Успех → редирект на `/dashboard`
   - Провал → сообщение об ошибке на той же странице

2. **Выход `/logout`**
   - Завершение сессии через `logout_user()`
   - Редирект на `/login`

3. **Модель пользователя в БД**
   ```
   users
   ├── id          INTEGER PRIMARY KEY
   ├── login       VARCHAR(64) UNIQUE NOT NULL
   ├── password    VARCHAR(256) NOT NULL   -- хеш bcrypt
   └── role        VARCHAR(32) NOT NULL    -- 'student' | 'curator' | 'teacher' | 'admin'
   ```

### Часть 2 - Авторизация

**Выбранный метод:** дискреционный способ с использованием списка прав доступа (ACL).

**Обоснование выбора:** ACL хорошо подходит для систем с небольшим числом чётко разграниченных ролей. Каждый маршрут явно объявляет, каким ролям он доступен.

**Что реализуется:**

1. **Декоратор `@requires_role`**
   - Проверяет роль текущего пользователя перед обработкой запроса
   - Если роль не разрешена → `HTTP 403 Forbidden`

2. **Таблица прав доступа (ACL)**

   | Ресурс (маршрут) | `student` | `curator` | `teacher` | `admin` |
   |-----------------|:---------:|:---------:|:---------:|:-------:|
   | `/dashboard` | ✅ | ✅ | ✅ | ✅ |
   | `/admin/users` | ❌ | ❌ | ❌ | ✅ |
   | `/profile/edit` | ✅ | ✅ | ✅ | ✅ |

3. **Страница ошибки 403** - понятное сообщение «Доступ запрещён»

### Часть 3 - Ролевое управление (RBAC)

**Выбранный метод:** статическое разделение обязанностей (один пользователь - одна роль).

**Обоснование выбора:** в системе управления практиками роли чётко разграничены и не пересекаются: студент не может быть одновременно куратором. Это делает статическую модель естественной и достаточной.

**Что реализуется:**

Личный кабинет `/dashboard` - **одна страница**, содержимое которой меняется в зависимости от роли.

#### Роль `student` видит:
- Своё ФИО, группу, направление подготовки
- Название организации и период практики
- Список назначенных заданий со статусом выполнения
- ФИО руководителя от кафедры и куратора от организации

#### Роль `curator` видит:
- Своё ФИО и организацию
- Список закреплённых студентов с отображением прогресса по заданиям

#### Роль `teacher` видит:
- Своё ФИО и кафедру
- Список студентов под руководством и их текущий статус по практике

#### Роль `admin` видит:
- Список всех пользователей системы
- Возможность изменить роль пользователя
- Возможность создать нового пользователя / деактивировать существующего

## 4. Структура файлов проекта

```
virtual_dekanat/
│
├── app.py                  # Точка входа, создание Flask-приложения
├── requirements.txt        # Зависимости
├── config.py               # Настройки (строка подключения к БД и т.п.)
│
├── models.py               # Модели SQLAlchemy (User)
│
├── auth/
│   └── routes.py           # /login, /logout
│
├── dashboard/
│   └── routes.py           # /dashboard - личный кабинет
│
├── access/
│   └── decorators.py       # Декоратор @requires_role для ACL
│
└── templates/
    ├── base.html            # Общий шаблон (navbar, footer)
    ├── login.html           # Страница входа
    ├── 403.html             # Страница ошибки доступа
    └── dashboard/
        ├── student.html     # Кабинет студента
        ├── curator.html     # Кабинет куратора
        ├── teacher.html     # Кабинет преподавателя
        └── admin.html       # Кабинет администратора
```
